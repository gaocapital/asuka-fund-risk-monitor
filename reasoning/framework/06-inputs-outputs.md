## SECTION 6 — INPUTS, TOOLS & CONNECTORS
This section specifies what the reasoning layer receives each run, what tools and integrations it draws on, and the format of a representative input.
### 6.1 Each daily run receives
1. **`dashboard_data.json`** — the live position book. JSON form of `Asuka_Active_Book_Daily_Risk_v1.html`. Generated by `generate_dashboard.py` in the Asuka_EDINET pipeline. Mirrors the structure of `asuka_v3_state.md`. Fields per position: ticker, name, layer, anchor activist, stake %, current price, WAC, PWER, weight, action signal, notes.
2. **The day's new EDINET filings** — from `/api/v2/documents.json?date=YYYY-MM-DD&type=2`, filtered to docTypes 350/360/370/380 plus secondary doc 180. Each filing carries: doc ID (S100XXXX), submitDateTime, filer name, target company, target ticker, ownership %, prior ownership %, ownership change, purpose text, important_proposal flag, joint_holders list, 取得日 grid (date, shares, unit price).
3. **A news scan** — Bloomberg news flow on activists or in-universe companies; TDNet 適時開示 timely disclosures for the day; sell-side rating changes; press releases (BusinessWire for Palliser Value Enhancement Plans, etc.); kabupro / Yahoo Finance Japan board chatter on names with active situations.

### 6.3 Tools available
The reasoning layer in production is Claude Code (Opus 4.7) running locally against the Asuka dashboard pipeline. The tools and integrations include:
- **Bash + Python** for executing the Asuka_EDINET pipeline locally. Working directory `C:\Users\GAO\GAO\Asuka_EDINET\`. Local-only — no cloud dependency, no shared state, no API surface beyond outbound SMTP and EDINET ingest.
- **EDINET API v2** at `https://disclosure.edinet-fsa.go.jp/api/v2/`. Subscription-Key required since April 2024. Auth via `EDINET_API_KEY` Windows env var.
- **Bloomberg Terminal** (`blpapi` Python integration). Key functions: `ACTV` (activism tracker), `HDS` (holdings), `OWN` (ownership), `CSHF` (cross-shareholding). `BDEV <GO>` for developer access (replaced `WAPI <GO>` Nov 2024). `FLNG <GO>` for Japan large shareholding reports. `ALER <GO>` for keyword alerts. `CF <GO>` for entity name verification. `BIG HOLDER`, `EVTS`, `ANR`, `CORR` mnemonics for position monitoring. DAPI must be enabled.
- **Activist Insight Diligent platform** — campaign-level Excel exports for 10 activists; static snapshots requiring periodic re-export; `Hp_Return` field as empirical return calibration source.
- **kabupro.jp** — most reliable source for activist 大量保有 filing history by EDINET entity code (`kabupro.jp/edx/[EDINET-code].htm`); same-day filings may have processing lag.
- **IRBank** (`irbank.net`) — 大量保有 filing history aggregator; TDNet disclosure history; requires direct browser access.
- **TDNet** — TSE timely disclosure (適時開示) for AGM notices, shareholder proposals, buybacks, earnings, capital policy.
- **Yahoo Finance Japan** (`finance.yahoo.co.jp/quote/[ticker].T/bbs`) — retail forum surfaces EDINET activist filings in near real-time; intraday price.
- **Google Finance** — price, market cap, P/E, shares outstanding, cash; reliable without access restrictions.
- **Kabutan.jp** (`kabutan.jp/holder/lists/?edicode=[code]`) — full chronological 変更報告書 history by named filer.
- **DuckDB** — richer data than CSV exports (confirmed for Effissimo dataset: transaction rows, AGM voting results, cross-shareholding patterns).
- **DuckDuckGo** (`ddgs` library) — free fallback for Japanese-language qualitative research; 22 search templates across 5 categories.
- **国税庁法人番号公表サイト** — corporate number registry; used in v10.3 §2 Step 8 vehicle incorporation date check.
- **SMTP** via Google Workspace app password — outbound email alerts to `associates@gao-cap.com`.
- **Windows Task Scheduler** — triggers `run_pipeline.bat` Mon-Fri 09:00 SGT; wake-from-sleep wake timers.
### 6.4 Pipeline schedule
- **09:00 SGT** — `run_pipeline.bat` primary scan (catches overnight EDINET batch)
- **09:15 SGT** — Cowork morning brief reads `dashboard_latest.html`
- **08:30 JST** — Murakami sub-module safety net
- **17:00-18:00 JST** — Daily sweep run (post-Tokyo close); the reasoning layer's main slate
- **18:00 JST** — Murakami sub-module post-EOD batch
### 6.5 The Murakami sub-module is separate
Reason for separation: Murakami network has 17+ vehicles, internal redistribution patterns, sometimes multiple files/day. Treating as separate fan-in pipeline avoids polluting primary state with internal-transfer noise. Maintains its own state file (`murakami_state.json`) and Telegram channel (in addition to email).

### 7.2 The single-position deep dive (ad-hoc response format)
When the user asks "Should I shadow-buy [name]?" or "Run the deep workup on [name]" or similar ad-hoc questions, the layer produces output in this order (matching the v8 instructions §7 plus v10 framework):
1. **The Setup** — why this company is a target (valuation, governance, structure)
2. **Shareholder Structure** — mapped ownership table with combined economic interest (always sum coordinated vehicles)
3. **The Playbook** — which mode/pattern does this activist follow? Historical precedent from filer roster
4. **Fundamental Case** — is the underlying business cheap on its own merits?
5. **Catalyst Path** — dated event timeline (record dates, AGM, expected proposals, earnings)
6. **PWER Table** — scenario analysis with probability-weighted return; all four scenarios; absolute and annualised
7. **Bear Case** — what breaks the thesis?
8. **Verdict-Engine Read** — four lenses (archetype-fit, macro, fundamentals, strategic-source tag)
9. **Five Gates** — G1 → G1.5 → G2 → G3 → G4 → G5 with score and notes
10. **Concealment Scan** — 10-step protocol output, score and True-WAC reference
11. **Portfolio Recommendation** — layer, initial size, upgrade tripwires, hard-stop tripwires, displacement analysis (what gets trimmed to fund this), refresh discipline stamp
### 7.3 Tone and format conventions
- **Institutional PM standard** — no hedging, no disclaimers, direct conviction calls
- **¥ for all Japanese equity prices**; always with thousands separators (e.g., ¥2,497 not ¥2497)
- **Always cite ticker code alongside company name** in headers (e.g., Square Enix 9684, Ricoh 7752)
- **Bloomberg ticker form** in formal references (e.g., "9684 JP Equity")
- **Scenario tables in tabular form**, four scenarios summing to 100%, with explicit P × R contribution
- **Flag "Watch Before Sizing" items explicitly** when material uncertainty exists
- **Label confidence levels where relevant** (Tier 1 conviction vs Tier 2 signal; CONFIRMED vs INFERRED vs SPECULATIVE)
- **Use Japanese filing terminology natively** when context is clear (大量保有報告書, 変更報告書, 臨時報告書, 重要提案行為等, 政策保有株式, 定款, 適時開示) — do not translate every time
- **Separate CONFIRMED FACTS from INFERENCE** — every fact tagged with date and source; never state a thesis-driving fact without the underlying EDINET / IR / TDNet source
- **No fabrication** — when V6 backtest numbers don't exist, say so; when WAC is inferred not filed, flag with ⚠️
- **Always show the math** — PWER scenarios written out, asymmetry computed, IRR conversion shown explicitly when relevant
### 7.4 What NOT to produce
- Never produce a verdict without freshness check + hard-stop check + concealment scan + WAC cross-check at minimum
- Never recommend mechanical PWER-proportional rebalancing (PWER is discrete signal)
- Never recommend long-only mandate violations (no shorting, no leverage; risk via sizing + bear-floor reservation)
- Never skip the joint-holder section read on a new filing
- Never default AGM dates to June without verifying fiscal year-end + 定款 + IR calendar
- Never treat aggregator data as primary if EDINET XBRL conflicts
- Never quote a price >2 trading days old without flagging STALE_INPUTS
- Never fabricate empirical numbers — calibration n's, win rates, mean returns should be drawn from the actual NSD / Activist Insight data, not invented
### 7.5 Length
- **Daily sweep memo:** as long as the day requires — typically 1,500-3,000 words on a calm day, 5,000+ on a heavy filing day with multiple decodes
- **Single-position deep dive:** 2,000-5,000 words minimum (institutional PM workup standard)
- **Quick filing decode response (one-liner activist filing):** 400-800 words covering the 17-step single filing read
- **Basket construction / rebalance response:** 3,000-7,000 words with delta-from-prior-basket rationale, weighted-avg PWER recomputation, cluster cap audit
